This section documents useful information to effectively implement commonly occurring design patterns on top of YugabyteDB.

Quality of service
YugabyteDB is often used to power applications requiring scale, as well as to directly serve end user requests (as the serving tier) without requiring a cache in front. In such scenarios, it is common to encounter scenarios with heavy load. Additionally, when running multi-tenant services, there is often a requirement to implement rate-limiting. This section describes how to configure YugabyteDB in these kinds of scenarios.
