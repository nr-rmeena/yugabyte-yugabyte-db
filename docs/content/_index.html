YugabyteDB Documentation

The Scalable Distributed SQL Database
Distributed SQL
/dəˈstribyo͞odəd ˈsēkwəl/
noun: distributed SQL database
  1. A distributed SQL database is a single relational database which replicates data across multiple servers.
  1. A distributed SQL database is a single relational database which replicates data across multiple servers.
    • YugabyteDB is a Distributed SQL database that is strongly consistent and supports consistency across racks, data centers, and wide area networks including cloud availability zones and cloud geographic zones.

verb: distributed SQL
  1. Distributing Data Storage
    Data including indexes are automatically distributed (aka sharded) across multiple nodes of the cluster, so that no single node becomes a bottleneck for ensuring high performance and high availability.
  2. Distributing Query Execution
    Queries are automatically distributed across multiple nodes of the cluster so that no single node becomes a bottleneck for query processing. Incoming queries are accepted by any node in the cluster.
Yugabyte DB
/yô,ge,bīt dē bē/
noun: Yugabyte DB
  1. The most popular distributed SQL database that is open-sourced under Apache v2.0 license
  1. The most popular distributed SQL database that is open-sourced under Apache v2.0 license
    • A high-performance distributed SQL database for powering global, internet-scale applications.
    • Built using a unique combination of high-performance document store, per-shard distributed consensus replication and multi-shard ACID transactions, YugabyteDB serves both scale-out RDBMS and internet-scale OLTP workloads with low query latency, extreme resilience against failures and global data distribution.
